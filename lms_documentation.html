🔑 Access Control Engine
Flexible
Permission System
EduNexus uses a layered permission architecture — role defaults → individual overrides → time gates → IP rules. Every module, feature, and data record can be access-controlled independently.
🔑
Three-Layer Model: (1) Role-level defaults set baseline access. (2) Individual user overrides can expand or restrict beyond role defaults. (3) Time/condition gates can add temporary or scheduled access rules.

⚡ Permission Rules Engine

📅 Time-Gated AccessGrant access to a module or content between specific dates. Automatically unlock results after review period ends.
🎯 Prerequisite-BasedUnlock a course, test, or module only after the user completes a prior requirement (course completion, score threshold, etc.).
🌍 IP / Location RestrictionRestrict module access to specific IP ranges or geographic regions (e.g., proctored exams only from campus network).
👤 Per-User OverrideAny role permission can be overridden for an individual user — grant or revoke access without changing the global role definition.
🔗 Course-Level AccessTutors can be assigned to specific courses only, limiting their visibility to enrolled students and their own course data.
🏷️ Batch PermissionsApply permission changes to an entire batch, cohort, or department in one action. Bulk enrollment + access grant.

Outsider / Guest / External User
Limited & Controlled Access
OUTSIDER

Outsiders are non-enrolled, external, or guest users. Their access is highly restricted by default and can be expanded for specific use cases: prospective students exploring the platform, auditors reviewing courses, parents monitoring progress, corporate clients accessing training, or industry mentors providing feedback.

⚠️
All outsider access is opt-in: By default, outsiders see nothing. Each level of access must be explicitly granted by Admin or Management. Guest links can be time-limited or revoked at any time.
🌍 Public / Guest Access 7 features ▶
  • Public Course Catalog
    Browse courses marked "Public" by the institution. View title, description, tutor info, syllabus, and reviews — without login.
    No LoginPublic Only
  • Demo / Preview Content
    Access designated preview materials for a course: first chapter, intro video, sample notes — as configured by instructor.
  • Demo Quiz
    Take a sample quiz/assessment created for prospective students to experience the platform.
  • Inquiry & Enrollment Request
    Submit an enrollment request or inquiry form. Receive automated response. Management reviews and grants full access.
  • Public Forum / Blog Read
    Read public discussion threads or knowledge-base articles if the institution enables public forum mode.
🔗 Invited / Granted Access (Custom) 9 features ▶
  • Parent Dashboard (Custom Role)
    When explicitly set up, parents can view their child's attendance, marks, progress, and upcoming schedule. Read-only. No course interaction.
    Read-OnlyChild-Linked
  • Corporate/Client Training Access
    Grant external corporate employees access to specific courses. Separate billing, progress tracking, and completion certificates.
  • Auditor View
    External academic auditors can be given read-only access to course content, assessments, and results for quality review.
  • Industry Mentor / Guest Lecturer
    Industry professionals invited as guest lecturers can host a meet session, upload a resource, or participate in a forum thread — within defined boundaries.
  • Observer Mode for Meets
    Invited outsiders can observe a live session as a silent participant (view-only, no mic/camera) if granted by host.
  • Time-Limited Guest Link
    Admins can generate guest access links valid for X hours/days. Link auto-expires. No registration required for basic access.

📅 Module
Attendance
System
Multi-modal attendance tracking — QR codes, geo-fencing, manual, biometric integration — with automated alerts, leave management, and compliance reporting.
📱
QR & OTP Check-In
Tutors generate time-limited QR codes or OTP links. Students scan/click to mark attendance. Anti-proxy validation.
🌍
Geo-Fence Attendance
Students must be within the defined geographic boundary (campus) to mark attendance. Admin sets allowed radius.
✋
Manual Override
Tutors and management can manually correct any attendance record with reason logging and audit trail.
📊
Reports & Heatmaps
Calendar heatmaps per student, subject-wise analytics, institution-wide trends, and exportable reports.
🔔
Auto-Alerts
Configurable thresholds trigger alerts to student, tutor, parent, and management when attendance falls low.
📝
Leave Management
Students apply for leave, attach documents. Approval workflow: tutor → management. Leave affects attendance report accordingly.
